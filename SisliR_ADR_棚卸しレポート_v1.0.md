# SisliR ADR番号体系 棚卸しレポート
## v1.0 (2026-06-13)

> **目的**: 全設計書ファイルからADR番号を全件抽出し、重複・欠番・命名衝突を分類・整理する。
> **対象ファイル**: files(100).zip内の全.mdファイル（20ファイル）
> **方針**: 重複の「性質」を3種に分類し、対処の優先度を決定する。

---

## 1. 重複の性質分類

重複には以下の3種類がある。対処が必要なのは **Type C のみ**。

| Type | 説明 | 対処 |
|------|------|------|
| **Type A** 参照重複 | 同一ADRを複数ファイルが「参照」している（定義は1箇所、他は言及） | 正常・対処不要 |
| **Type B** コピー重複 | 旧バージョンファイルと現バージョンが同じADRを持つ（ファイル世代の重複） | 旧バージョンの整理で解消 |
| **Type C** 意味衝突 | **同じ番号に異なる内容の決定が定義されている**（真の重複バグ） | **要リナンバリング** |

## 2. Type C（意味衝突）— 対処必須

### 🔴 ADR-161: 3件の意味衝突

| ファイル | ADR-161の内容 |
|---------|--------------|
| **partMM v5.1** | `gradeLevel`をPart OOのOrder.gradeと共通型に統一（'basic'→'basic3d'） |
| **partQQ** | Higgsfieldを従量課金から月額サブスク（Pro/Ultimate）へ移行し、クレジット残量を日次監視 |
| **partRR** | 同上（QQと同一内容だがRRで再定義） |
| **CLAUDE.md** | 上記運用ポリシー（ADR-161/ADR-162・Part RR参照）として記述 |

**リナンバリング案**:
```

  ADR-168: Higgsfieldサブスク移行・クレジット日次監視         ← 新番号（partQQ/RR/CLAUDE.md更新）
```
